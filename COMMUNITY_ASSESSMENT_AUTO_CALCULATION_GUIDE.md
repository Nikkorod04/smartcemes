# Community Assessment Auto-Calculation System
## Implementation Guide & Usage

**Created:** March 25, 2026  
**Status:** Ready for Implementation  
**Trigger:** Automatic on each assessment submission

---

## Overview

This system automatically calculates community-level averages and statistics from individual needs assessment responses and displays them on the communities management page.

**Key Feature:** Every time an assessment is submitted/updated, the system recalculates all community statistics in real-time.

---

## System Components

### 1. **Database Tables**

#### `community_needs_assessments` (Existing)
- Stores individual respondent data (9 sections, 50+ fields)
- Updated when secretary enters assessment
- Observer triggers calculation on save

#### `community_assessment_summaries` (New)
- Stores calculated statistics per community
- Auto-created/updated on first assessment
- Contains 40+ calculated metrics

### 2. **Models**

#### CommunityNeedsAssessment
- Has relationships to User and Community
- Methods: `isSubmitted()`, `getRespondentFullName()`
- Scopes: `submitted()`, `forCommunity()`

#### CommunityAssessmentSummary
- Relationship to Community
- Stores all calculated metrics
- Auto-generated by calculation service

#### Community
- New relationship: `assessmentSummary()`
- New relationship: `needsAssessments()`
- Eager-loads summary in queries

### 3. **Services**

#### CommunityAssessmentCalculationService
- **Method:** `calculateSummary($communityId)`
- **Trigger:** Called automatically on every assessment change
- **Processes:**
  - Numeric averages (age, ratings)
  - Modal values (most common selections)
  - Percentages (yes/no fields)
  - Top problems/themes
  - Service satisfaction ratings

### 4. **Observer Pattern**

#### CommunityNeedsAssessmentObserver
- Registered in `AppServiceProvider`
- Listens to create/update/delete events
- Automatically recalculates summaries
- **No manual triggers needed**

### 5. **UI Component**

#### `community-assessment-summary` Blade Component
- Displays summary card with key metrics
- Shows top problems and themes
- Includes last calculation timestamp
- Shows response count

## How It Works

### Step-by-Step Flow

```
1. Secretary enters Community Needs Assessment (F-CES-001)
        ↓
2. Form submitted, assessment record created in DB
        ↓
3. CommunityNeedsAssessmentObserver.created() fires
        ↓
4. Assessment status = 'submitted' ✓
        ↓
5. CommunityAssessmentCalculationService.calculateSummary() called
        ↓
6. Service queries all submitted assessments for community
        ↓
7. Calculates:
   - Averages (age, ratings)
   - Modals (most common values)
   - Percentages (yes/no counts)
   - Top problems (frequency sort)
   - Key themes (keyword extraction)
        ↓
8. Finds or creates CommunityAssessmentSummary record
        ↓
9. Updates all calculated fields
        ↓
10. Sets last_calculated_at timestamp
        ↓
11. On Communities page:
    a) Community loads via ManageCommunities component
    b) assessmentSummary relationship eager-loaded
    c) Summary component renders stats card
    d) Card displays in expandable row below community
```

### Trigger Events

The calculation is triggered **automatically** when:

✅ New assessment is created (status = submitted)  
✅ Assessment status changes to submitted  
✅ Assessment data is edited (any assessment field changes)  
✅ Assessment is deleted (recalculates remaining)  
✅ Assessment is restored from soft delete  

❌ Does NOT recalculate for draft assessments

## Using the Component

### Display Summary on Any Page

```blade
<x-community-assessment-summary :summary="$community->assessmentSummary" />
```

### Access Summary Data in Code

```php
// Get community with summary
$community = Community::with('assessmentSummary')->first();

// Access metrics
$summary = $community->assessmentSummary;

echo $summary->avg_age;                              // 35.5
echo $summary->percent_interested_in_training;     // 75
echo $summary->overall_service_satisfaction;       // 3.8
echo $summary->top_family_problems;                // array of top 5
echo $summary->total_assessments;                  // Total responses
echo $summary->last_calculated_at;                 // Carbon timestamp
```

### Example: Display on Dashboard

```blade
@foreach($communities as $community)
    <div class="community-card">
        <h3>{{ $community->name }}</h3>
        
        @if($community->assessmentSummary)
            <p>Avg Age: {{ $community->assessmentSummary->avg_age }}</p>
            <p>Responses: {{ $community->assessmentSummary->total_assessments }}</p>
        @else
            <p>No assessments yet</p>
        @endif
    </div>
@endforeach
```

## Database Reset & Recalculation

### If You Need to Recalculate All Summaries

Create an Artisan command:

```php
// app/Console/Commands/RecalculateAssessmentSummaries.php
php artisan make:command RecalculateAssessmentSummaries
```

```php
public function handle()
{
    $communities = Community::all();
    $service = app(CommunityAssessmentCalculationService::class);
    
    foreach ($communities as $community) {
        $service->calculateSummary($community->id);
        $this->info("Calculated: {$community->name}");
    }
}
```

Run with: `php artisan recalculate:assessments`

## Performance Considerations

### Queries per Call
- **Calculation:** 1 query to fetch submitted assessments
- **Update:** 1 query to update or create summary
- **Display:** Eager-loaded via `.with('assessmentSummary')`

### Optimization Tips
```php
// Good: Eager load when you need summaries
Community::with('assessmentSummary')->get();

// Avoid: N+1 queries
Community::all(); // then accessing $community->assessmentSummary
```

### When Many Students Assess Same Community
- Calculation happens in-memory (fast)
- Database write is single update
- No performance degradation with 100+ assessments per community

## Troubleshooting

### Summary Not Calculating
**Check:**
1. Assessment status = 'submitted' (not 'draft')
2. Observer registered in `AppServiceProvider.php`
3. Cache cleared: `php artisan cache:clear`

### Summary Shows Old Data
**Solution:**
```php
// Force recalculate single community
$service = app(CommunityAssessmentCalculationService::class);
$service->calculateSummary($community_id);
```

### Calculation Takes Too Long
**If 1000+ assessments:**
- Move calculation to queue job
- Update observer to dispatch job instead of sync call
